# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**Memory Game** - 子供向けオフライン神経衰弱アプリ

### プロダクトコンセプト
- **子供が一人で始めやすい**: 文字説明に頼らず、見た目とタップだけで遊び始められる
- **短時間で達成感がある**: 1ゲーム3分以内を目安に、成功体験をすぐ得られる
- **安心して渡せる**: オフライン完結、広告なし、課金なしで親が安心して使える

### プロダクトビジョン
5歳前後の子供が、親の細かな説明なしでも短時間で楽しく遊べる記憶カードゲームを提供する。  
明るいアニメーション、音、振動を組み合わせて、成功体験を分かりやすく伝える。  
同時に、React Native + Expo を使ったモバイルアプリ開発と、Codex主体のスペック駆動開発を実践できる土台にする。

### 目的
- 子供が短時間で繰り返し遊びたくなるシンプルな神経衰弱アプリを提供する
- 親が安心してスマートフォンを渡せる、オフライン完結の体験を作る
- モバイルアプリ開発と Agent 主体の開発フローを学べる実践プロジェクトにする

## ターゲットユーザー

### プライマリーペルソナ: 5歳前後の子供
- 文字を読むより、絵や動きで理解する
- タップ中心の直感的な操作を好む
- 1回3分程度の短いプレイに集中しやすい
- 失敗時に強いストレスを感じる演出は避けたい
- 「カードをめくる」「同じ絵を見つける」という明快な目標だと継続しやすい

### セカンダリーペルソナ: 親・保護者
- 数分間だけ子供に安全に遊ばせたい
- オンライン接続や課金、広告を気にせず使いたい
- すぐ遊び始められ、説明コストが低いアプリを求めている
- 音や振動があっても、操作はシンプルであってほしい

## 成功指標 (KPI)

### プライマリーKPI
- 初回プレイ観察で、子供がホーム画面表示から30秒以内に最初のカードをめくれる割合: 80%以上
- 4 x 5盤面のMVPゲームで、開始からクリアまたは自発的なリスタートまでが3分以内に収まるセッション割合: 90%以上
- 内部QAおよび配布前検証でのクラッシュフリーセッション率: 99%以上

### セカンダリーKPI
- 機内モード検証で、起動からクリアまで主要フローが問題なく動作する割合: 100%
- 保護者ヒアリングでの「安心して子供に渡せる」評価: 5段階中4以上
- カードタップから視覚フィードバック開始までの応答時間: 100ms以内

## 機能要件

### コア機能 (MVP)

#### 1. ゲーム開始とリスタート

**ユーザーストーリー**:  
子供または親として、すぐに遊び始めたり遊び直したりするために、1タップで新しいゲームを開始・再開始したい

**受け入れ条件**:
- [ ] アプリ起動時に、遊び始めるための主要アクションがすぐ分かるホーム表示になる
- [ ] 新しいゲーム開始時に、10ペア・計20枚のカードがシャッフルされた4 x 5盤面を生成する
- [ ] プレイ中およびゲーム終了後に、リスタート操作ができる
- [ ] リスタート時に `cards`、`selectedCardIds`、`matchedPairs`、`gameStatus` が初期状態に戻る

**優先度**: P0 (必須)

#### 2. カードめくりとマッチ判定

**ユーザーストーリー**:  
子供として、同じ絵柄のカードを探すために、カードをめくって結果をすぐ知りたい

**受け入れ条件**:
- [ ] 未マッチの裏向きカードをタップすると、そのカードだけが表向きになる
- [ ] 同時に表向きにできる未確定カードは最大2枚までである
- [ ] 2枚の絵柄が一致した場合、その2枚は `isMatched = true` となり表向きのまま残る
- [ ] 2枚の絵柄が不一致の場合、0.8秒以上1.2秒以下の遅延後に裏向きへ戻る
- [ ] 2枚の判定中は追加タップを受け付けない

**優先度**: P0 (必須)

#### 3. 演出・音・振動フィードバック

**ユーザーストーリー**:  
子供として、うまくできたかを直感的に理解するために、視覚・音・振動で反応がほしい

**受け入れ条件**:
- [ ] カードをめくった瞬間に視覚的な反応がある
- [ ] マッチ成功時に、成功を強調するアニメーションと音、振動の少なくとも2種類が発火する
- [ ] ミスマッチ時は、失敗を責めない穏やかなフィードバックのみを返す
- [ ] 全ペア揃った時に、ゲームクリア演出が再生される

**優先度**: P0 (必須)

#### 4. 子供向けUI

**ユーザーストーリー**:  
子供として、文字を読まなくても遊べるように、大きく分かりやすい画面で操作したい

**受け入れ条件**:
- [ ] 主要ボタンとカードは、5歳前後の子供が押しやすい十分なタップ領域を持つ
- [ ] 主要な操作はアイコン、色、配置で理解でき、長い説明文を必要としない
- [ ] 画面は縦向きで完結し、1画面内で主要プレイ体験が終わる
- [ ] 色以外の手段でも状態が識別できるよう、アニメーションや形状差を併用する

**優先度**: P0 (必須)

### 将来的な機能 (Post-MVP)

#### 難易度切り替え
- 4 x 5以外の盤面サイズを選べるようにする
- **優先度**: P1 (重要)

#### ローカル成績保存
- ベストタイムやプレイ回数を端末内に保存する
- **優先度**: P1 (重要)

#### テーマ切り替え
- カード絵柄や色テーマを選べるようにする
- **優先度**: P2 (できれば)

#### 保護者向け設定
- 音量や振動の有無を保護者が切り替えられるようにする
- **優先度**: P2 (できれば)

## 非機能要件

### パフォーマンス
- アプリ起動から最初の操作可能状態まで: 2秒以内を目標とする
- カードタップから反応開始まで: 100ms以内
- ミスマッチ判定を含む1ターンの解決時間: 1.2秒以内
- 中価格帯スマートフォンで、プレイ中に体感的なフレーム落ちが継続しないこと

### ユーザビリティ
- 文字が読めなくても、ホーム画面からプレイ開始まで到達できる
- 子供が誤操作しにくい大きさと余白を確保する
- 失敗時にも再挑戦したくなる穏やかな演出にする

### 信頼性
- インターネット接続がなくても起動からゲーム終了まで遊べる
- タップ連打や素早い入力でも、カード状態が破綻しない
- 音声・振動が使えない端末でもゲーム進行は継続できる

### セキュリティ / プライバシー
- ユーザーアカウント、個人情報入力、外部送信をMVPでは持たない
- 広告SDK、課金SDK、トラッキングSDKを導入しない
- カメラ、マイク、位置情報などゲームに不要な権限を要求しない

### 保守性
- ゲームロジックはUIや端末APIから分離し、単体テスト可能にする
- MVPの主要ゲームルールは unit test、主要画面フローは component test、音・振動・オフライン挙動は iOS / Android 各1実機で検証できるようにする
- 用語、状態、盤面仕様は `docs/` の恒久ドキュメントに合わせて管理する
- 新機能追加時は `.steering/` を更新し、ドキュメントと実装の差分を追跡可能にする

## スコープ外

明示的にスコープ外とする項目:
- オンライン対戦
- ユーザーアカウント
- 広告表示
- 課金
- 外部ランキング
- サーバー連携
- プレイデータのクラウド保存
