# プロジェクト概要

子供向けのシンプルな記憶カードゲーム（神経衰弱）を作る。

React Native + Expo を使ったモバイルアプリとして実装し、
インターネット接続がない環境でも遊べることを前提とする。

小さな子供でも遊べるように、次のフィードバックを重視する。

- 視覚的に分かりやすいUI
- シンプルな操作
- アニメーション
- 音
- 振動

# ゲーム基本ルール

神経衰弱ゲームをベースとする。

基本ルール:

1. カードは裏向きで配置される
2. プレイヤーは1回に2枚カードをめくる
3. 同じ絵柄ならカードは表のまま残る
4. 違う絵柄ならカードは元に戻る
5. すべてのペアが揃ったらゲーム終了

---

# ゲーム状態（Game State）

ゲームの状態は次のデータで管理する。

## cards

カードの一覧。

各カードは以下の情報を持つ。

- id
- type（絵柄）
- isFlipped
- isMatched

例:

```
cards = [
  { id: 1, type: "unicorn", isFlipped: false, isMatched: false },
  { id: 2, type: "rainbow", isFlipped: false, isMatched: false }
]
```

---

## selectedCards

現在めくられているカード。
最大2枚。

```
selectedCards = [cardId1, cardId2]
```

---

## matchedPairs

揃ったペア数。

```
matchedPairs = number
```

---

## gameStatus

ゲームの状態。

```
"idle"      // ゲーム開始前
"playing"   // プレイ中
"finished"  // クリア
```

# ゲーム盤面仕様（初期案）

子供向けの難易度を考慮し、盤面サイズは以下を検討する。

候補:

- 4 x 5（20枚 / 10ペア）
- 5 x 6（30枚 / 15ペア）

初期MVP案:

**4 x 5 を採用する可能性が高い。**

理由:

- プレイ時間が短くなる
- 小さい子供でも達成できる
- UIがシンプルになる

将来的には難易度として盤面サイズを変更できる可能性がある。

# Repository Structure 方針（初期案）

このプロジェクトでは、学習目的のため
**シンプルで理解しやすいディレクトリ構造**を優先する。

過度なレイヤー分割は行わない。

## 基本方針

- 初学者でも理解しやすい構造
- React Nativeの基本構造に近い
- ファイル数を最小限にする
- 機能単位で整理する

## 想定ディレクトリ構造

```
src
 ├ App.js
 ├ screens
 │   └ GameScreen.js
 │
 ├ components
 │   ├ Card.js
 │   └ Board.js
 │
 ├ game
 │   ├ createDeck.js
 │   ├ shuffleDeck.js
 │   └ checkMatch.js
 │
 └ assets
     ├ images
     └ sounds
```
